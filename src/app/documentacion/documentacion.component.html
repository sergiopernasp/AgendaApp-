Estructura de componentes:

AppComponent: Este componente es el componente principal de la aplicación. Contiene la lógica principal y la estructura de la página principal. A través de la variable componenteActual, se define el componente actual que se muestra en función de la selección del menú. Además, se utiliza la variable listaTareas para almacenar las tareas agregadas.

Menu: Este componente representa el menú de navegación de la aplicación. Contiene una lista de elementos li; que representan las diferentes opciones del menú. Al hacer clic en un elemento del menú, se emite un evento componenteSeleccionado con el nombre del componente seleccionado.

Tarea: Este componente representa una tarea individual en la lista de tareas. Recibe las propiedades de la tarea (titulo, fecha, descripcion, estado). El componente también emite eventos cuando se hace clic en el botón "Borrar" o cuando se cambia el estado de la tarea.

AgregarTarea: Este componente representa el formulario para agregar una nueva tarea. Los datos del formulario (titulo, fecha, hora, descripcion, estado) se vinculan a las variables correspondientes utilizando [(ngModel)]. Cuando se envía el formulario, se emite un evento tareaAgregada con la nueva tarea como argumento.

TareasPorFecha: Este componente muestra una lista de tareas ordenadas cronológicamente. Puede alternar entre mostrar todas las tareas y mostrar solo las tareas pendientes. Las tareas se obtienen a través de la variable listaTareas, que se recibe como una propiedad Input.

TareasPorEstado: Este componente muestra una lista de tareas separadas por estado (pendientes y realizadas). Al igual que el componente anterior, utiliza la variable listaTareas para obtener las tareas.

Estructura de código y las invocaciones utilizadas en cada componente de tu aplicación:

appComponent:

  • Invocaciones:
    • @Component: Se utiliza para decorar la clase AppComponent y definir las propiedades del componente, como el selector (selector), la plantilla (templateUrl) y los estilos (styleUrls).
    • componenteActual: string: Esta variable se utiliza para almacenar el nombre del componente actual que se muestra en función de la selección del menú.
    • listaTareas: Tarea[]: Es un array que almacena las tareas agregadas.
    • app-menu: Invoca al componente menu para mostrar el menú de navegación.

menu:

  • Invocaciones:
    • componenteSeleccionado: Se utiliza para emitir un evento cuando se selecciona un componente del menú.
    • (click)="seleccionarComponente(componente)": Se utiliza en cada elemento del menú para llamar a la función seleccionarComponente cuando se hace clic en él y pasar el nombre del componente seleccionado como argumento.

tarea:

  • Invocaciones:
    • @Input: Se utiliza para recibir las propiedades de la tarea (titulo, fecha, descripcion, estado) desde el componente padre.
    • @Output: Se utiliza para emitir eventos cuando se hace clic en el botón "Borrar" o cuando se cambia el estado de la tarea.
    • (click)="borrarTarea()": Se utiliza para llamar a la función borrarTarea cuando se hace clic en el botón "Borrar".
    • (change)="cambiarEstado()": Se utiliza para llamar a la función cambiarEstado cuando se cambia el estado de la tarea.

agregarTarea:

  • Invocaciones:
    • tareaAgregada: Se utiliza para emitir un evento cuando se agrega una nueva tarea.
    • [(ngModel)]: Se utiliza para vincular los datos del formulario (titulo, fecha, hora, descripcion, estado) a las variables correspondientes.

tareasPorFecha:

  • Invocaciones:
    • @Input: Se utiliza para recibir la lista de tareas desde el componente padre.
    • tareasFiltradas: Es un array que almacena las tareas filtradas por fecha.

tareasPorEstado:

  • Invocaciones:
    • @Input: Se utiliza para recibir la lista de tareas desde el componente padre.
    • tareasFiltradas: Tarea[]: Es un array que almacena las tareas filtradas por estado.
    • [(ngModel)]="estadoSeleccionado": Se utiliza para crear un campo de selección donde se puede elegir un estado de tarea.
    • (ngModelChange)="filtrarTareasPorEstado()": Se utiliza para llamar a la función filtrarTareasPorEstado cuando se cambia el estado seleccionado.
    • filtrarTareasPorEstado(): Es una función que filtra las tareas en función del estado seleccionado y actualiza el array tareasFiltradas con las tareas correspondientes.
    • tareasFiltradas.length: Se utiliza para mostrar el número de tareas filtradas en la plantilla.

Problemas y mejoras

Por último añadir que el mayor problema con el que me he enfrentado y que al principio no entendía como hacer ha sido el pasar variables de un componente a otro. Por ejemplo pasar la lista de tareas desde app hasta el listado. Al final me he decantado por recibirlo como un Input dentro del listado. Otro problema relacionado con lo mismo ha sido el modificar el estado de una tarea o eliminarla. Para este problema he usado un eventEmiter para hacerle llegar al padre los cambios necesarios. He intentado añadir la modificación de tareas pero no he sido capaz, ya que cuando modificaba una acababa creando una nueva, asique finalmente lo he eliminado.
